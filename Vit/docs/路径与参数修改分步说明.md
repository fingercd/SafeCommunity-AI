# 路径与超参数修改分步说明

## 原则说明

- **路径**：你的视频、CSV、输出目录放在哪里，就改对应项；不改则用脚本内置默认（相对项目根）。
- **超参数**：影响效果/速度的数值；有配置文件的步骤优先改 YAML，没有的再改命令行里那几项。
- **运行方式**：在**项目根目录**（即包含 `lab_anomaly` 和 `lab_dataset` 的那一层）打开终端，再执行下面给出的命令。

---

## 步骤 1：扫描视频，生成标签表 CSV

**跑的脚本：** [lab_anomaly/data/index_build.py](../lab_anomaly/data/index_build.py)

**这些参数是什么：**

| 参数 | 含义 | 默认值（你不写就按这个来） |
|------|------|----------------------------|
| `--dataset_root` | 数据集根目录（用来算视频相对路径） | `lab_dataset` |
| `--videos_root` | 要扫描的视频所在文件夹 | `lab_dataset/raw_videos` |
| `--out_csv` | 生成的 CSV 保存路径 | `lab_dataset/labels/video_labels.csv` |
| `--default_label` | 新扫到的视频默认标签 | `unknown` |

**一步一步做：**

1. **准备文件夹**
   - 在项目根下建好：`lab_dataset/raw_videos`（或你打算放视频的目录）。
   - 视频建议按：`raw_videos/{摄像头名}/{日期YYYYMMDD}/xxx.mp4` 放，例如 `raw_videos/cam0/20260204/a01.mp4`。

2. **决定要不要改路径**
   - 视频就在 `lab_dataset/raw_videos`、CSV 想生成到 `lab_dataset/labels/video_labels.csv`：**不用改**，直接运行下面「3」里的命令。
   - 视频在别处（例如 `D:/my_videos`）：把下面命令里的 `lab_dataset/raw_videos` 换成 `D:/my_videos`（`--videos_root D:/my_videos`），同时 `--dataset_root` 要能包含这个目录或与之一致（若用默认 `lab_dataset`，则 `--out_csv` 一般保持 `lab_dataset/labels/video_labels.csv` 即可）。

3. **运行（二选一）**
   - 用默认路径（视频在 `lab_dataset/raw_videos`）：
     ```bash
     python -m lab_anomaly.data.index_build
     ```
   - 自定义视频目录时（把 `你的视频目录` 换成实际路径）：
     ```bash
     python -m lab_anomaly.data.index_build --videos_root 你的视频目录
     ```

4. **之后**
   - 用 Excel 或记事本打开生成的 `lab_dataset/labels/video_labels.csv`，把要参与训练的行的 `label` 改成 `normal`（或具体异常类名），保存。

---

## 步骤 2：手动改标签

不运行任何脚本。只改上一步得到的 `lab_dataset/labels/video_labels.csv` 里的 `label` 列（至少有一批是 `normal`）。

---

## 步骤 3：提取 ViT 特征（embedding）

**跑的脚本：** [lab_anomaly/train/extract_embeddings.py](../lab_anomaly/train/extract_embeddings.py)  
**用的配置：** [lab_anomaly/configs/embedding_example.yaml](../lab_anomaly/configs/embedding_example.yaml)（推荐改这个，不用记一长串命令）

**这些项是什么、在哪改：**

- **路径（YAML 顶层的键）：**
  - `dataset_root`：数据集根目录，默认 `lab_dataset`。
  - `labels_csv`：上一步的 CSV，默认 `lab_dataset/labels/video_labels.csv`。
  - `out_dir`：embedding 输出目录，默认 `lab_dataset/derived/embeddings`。  
  若你 CSV 或视频不在默认位置，就改这三个。

- **超参数（YAML 里 `sampling` 和 `runtime`）：**
  - `sampling.clip_len`：每个片段的帧数，默认 16。
  - `sampling.frame_stride`：帧步长，默认 2。
  - `sampling.num_clips_per_video`：每条视频采多少个片段，默认 8（可调多/少）。
  - `runtime.batch_size`：批大小，默认 4（显存小可改小）。
  - `runtime.limit`：只处理前 N 个 clip，0 表示不限制。

- **模型/设备（YAML 里 `encoder`）：**
  - `encoder.model_name`：ViT 模型名，默认 `MCG-NJU/videomae-base`，一般不用改。
  - `encoder.device`：`auto` / `cuda` / `cpu`。
  - `encoder.use_half`：是否用半精度，默认 true。

**一步一步做：**

1. 用记事本或 VS Code 打开 `lab_anomaly/configs/embedding_example.yaml`。
2. 若你的 CSV 或视频不在默认路径，改 `dataset_root`、`labels_csv`、`out_dir`。
3. 若要调效果/速度，改 `sampling` 下的 `clip_len`、`frame_stride`、`num_clips_per_video` 以及 `runtime.batch_size`。
4. 保存 YAML。
5. 在项目根运行：
   ```bash
   python -m lab_anomaly.train.extract_embeddings --config lab_anomaly/configs/embedding_example.yaml --save_format npy_per_clip
   ```
   （`--save_format npy_per_clip` 建议保留，后面开放集脚本默认用这种格式。）

## 步骤 4：训练开放集异常检测（KMeans + One-Class SVM）

**跑的脚本：** [lab_anomaly/train/fit_kmeans_ocsvm.py](../lab_anomaly/train/fit_kmeans_ocsvm.py)  
**没有 YAML**，路径和超参数都在命令行里，但都有默认值。

**这些参数是什么、默认值：**

| 参数 | 含义 | 默认值 |
|------|------|--------|
| `--embeddings_dir` | 上一步生成的 embedding 目录 | `lab_dataset/derived/embeddings` |
| `--out_dir` | 本步输出目录（模型和阈值） | `lab_dataset/derived/open_set` |
| `--normal_label` | CSV 里「正常」的标签名 | `normal` |
| `--k` | KMeans 聚类个数 | 16 |
| `--nu` | One-Class SVM 的 nu | 0.05 |
| `--quantile` | 用 normal 的多少分位数当阈值（如 0.95=95%） | 0.95 |
| `--min_cluster_size` | 簇内样本少于此数不单独训 OCSVM | 20 |
| `--gamma` | OCSVM 核参数 | scale |
| `--limit` | 只用前 N 条 normal clip（0=不限制） | 0 |

**一步一步做：**

1. 若 embedding 和输出都用默认目录，**只跑**：
   ```bash
   python -m lab_anomaly.train.fit_kmeans_ocsvm
   ```
2. 若只改超参数（例如聚类数、阈值分位）：
   ```bash
   python -m lab_anomaly.train.fit_kmeans_ocsvm --k 20 --quantile 0.95
   ```
   （按需改 `--k`、`--nu`、`--quantile` 等。）
3. 若 embedding 或输出目录不在默认路径，加上路径：
   ```bash
   python -m lab_anomaly.train.fit_kmeans_ocsvm --embeddings_dir 你的embedding目录 --out_dir 你的输出目录
   ```
   再按需加 `--k`、`--quantile` 等。

## 步骤 5：推理（实时 RTSP 或本地视频）

**跑的脚本：** [lab_anomaly/infer/rtsp_service.py](../lab_anomaly/infer/rtsp_service.py)  
**用的配置：** [lab_anomaly/configs/rtsp_service_example.yaml](../lab_anomaly/configs/rtsp_service_example.yaml)

**这些项是什么、在哪改：**

- **必改（YAML）：**
  - `rtsp_url`：RTSP 地址或本地视频路径，例如 `rtsp://用户:密码@IP:554/通道` 或 `D:/test.mp4`。
  - `artifacts.open_set_dir`：上一步输出目录，默认 `lab_dataset/derived/open_set`。
  - 若用已知分类器：`artifacts.known_checkpoint` 指向 `checkpoint_best.pt` 路径。

- **可选（YAML）：**
  - `sampling.clip_len`、`frame_stride`、`sample_fps`、`window_stride`：与训练时一致或按需微调。
  - `fusion.min_known_prob`：已知类置信度下限。
  - `runtime.cooldown_sec`、`min_consecutive`：报警冷却、连续几帧才报警。
  - `outputs.save_snapshot`、`api_url`：是否截图、是否上报 API。

**一步一步做：**

1. 复制并编辑配置（或直接改示例）：打开 `lab_anomaly/configs/rtsp_service_example.yaml`。
2. 改 `rtsp_url` 为你的 RTSP 或本地 mp4 路径。
3. 改 `artifacts.open_set_dir` 为 `lab_dataset/derived/open_set`（或你步骤 4 的 `--out_dir`）。
4. 按需改 `sampling`、`fusion`、`runtime`、`outputs`。保存。
5. 在项目根运行：
   ```bash
   python -m lab_anomaly.infer.rtsp_service --config lab_anomaly/configs/rtsp_service_example.yaml
   ```
   若不想用配置文件，可命令行直接指定流地址和开放集目录：
   ```bash
   python -m lab_anomaly.infer.rtsp_service --rtsp "你的RTSP或本地视频路径" --open_set_dir lab_dataset/derived/open_set
   ```

---

## 小结：改路径 / 超参数时改哪里

- **步骤 1**：路径在命令行里改（或用默认）；无配置文件。
- **步骤 3**：路径和采样/批大小等 → 改 [embedding_example.yaml](../lab_anomaly/configs/embedding_example.yaml)。
- **步骤 4**：路径和 k、nu、分位数等 → 在运行 `fit_kmeans_ocsvm` 时加对应 `--参数名`。
- **步骤 5**：流地址、开放集目录、采样与报警相关 → 改 [rtsp_service_example.yaml](../lab_anomaly/configs/rtsp_service_example.yaml)。

这样你可以按「先改哪个文件/哪几项、再运行哪条简短命令」一步一步做，而不用背一长串完整命令。
